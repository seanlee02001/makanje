# MakanJe ‚Äî Family Meal Plan App Design

**Date:** 2026-03-15
**Status:** Approved

---

## Context

MakanJe is a family meal planning web app (MVP, family-only). The problem: families waste time and mental energy deciding what to eat each week, and grocery shopping is disorganized. The app solves this with a weekly meal planning grid, a shared meal library, and an auto-generated editable shopping list. It's a mobile-accessible PWA so family members can use it on their phones without installing anything from an app store.

---

## Stack

- **Frontend + Backend:** Next.js 14 (App Router) + TypeScript
- **Database + Auth + Realtime:** Supabase (Postgres + Supabase Auth + Realtime subscriptions)
- **Styling:** Tailwind CSS (mobile-first)
- **Deployment:** Vercel
- **PWA:** next-pwa for home screen installability
- **Recipe scraping:** `@extractus/article-parser` or `recipe-scraper` npm package

---

## Data Model (Postgres via Supabase)

```sql
families (id, name, invite_code)

users (id, email, name, family_id)

meals (id, family_id, name, source_url, created_by, created_at)

ingredients (id, meal_id, name, quantity, unit)

meal_plan_slots (
  id, family_id, week_start_date,
  day ENUM(mon,tue,wed,thu,fri,sat,sun),
  meal_slot ENUM(breakfast,lunch,dinner),
  meal_id
)

shopping_lists (
  id, family_id, week_start_date,
  items JSONB  -- [{ name, quantity, unit, checked, source }]
)
```

Row-Level Security (RLS) on all tables scoped to `family_id` so users only see their own family's data.

## Key Features & Flows

### 1. Family Onboarding
- Sign up ‚Üí create family (generates `invite_code`) OR enter existing invite code
- Invite link shared via `/settings` ‚Äî anyone with the link can join

### 2. Weekly Meal Plan Grid
- 7-column grid (Mon‚ÄìSun), configurable slots per day (user taps to enable/disable breakfast, lunch, dinner per day)
- Empty slot ‚Üí tap ‚Üí modal to pick from meal library or navigate to add new
- Slot filled ‚Üí tap ‚Üí replace or clear
- Realtime: Supabase Realtime subscription updates all family members' views live

### 3. Meal Library
- List/grid view of all family meals
- Manual add: name + ingredients (name, qty, unit per line)
- URL import: paste URL ‚Üí API route scrapes title + ingredients ‚Üí user reviews & edits ‚Üí saves

### 4. Shopping List
- "Generate Shopping List" button on `/plan`
- Aggregates all ingredients from planned meals for the week
- Deduplicates by ingredient name (merges quantities)
- Stored in `shopping_lists` table as editable JSONB
- Family members can check off items and add manual extras
- Realtime sync so two people shopping together see same state

### 5. Recipe Scraping
- `POST /api/import-recipe` accepts `{ url }`
- Server-side fetch + parse with `recipe-scraper` or `@extractus/article-parser`
- Returns `{ title, ingredients[] }` ‚Äî user reviews before saving
- On parse failure: return whatever was extracted + let user fill in the rest manually

## Error Handling

| Scenario | Handling |
|---|---|
| Recipe URL won't parse | Show partial data, user edits manually |
| Two users edit same slot simultaneously | Last write wins (sufficient for family scale) |
| Duplicate ingredients in shopping list | Merged by name during generation |
| Offline | PWA caches current week's plan for read-only |

## MVP Scope

### In Scope
- Family auth + invite code join
- Weekly meal plan grid with configurable slots
- Meal library (manual + URL import)
- Shopping list (auto-generated, editable, real-time checkboxes)
- PWA installable on mobile

### Out of Scope (Post-MVP)
- AI meal suggestions
- Nutritional info
- Push notifications
- Multiple family groups per user
- Ratings / favorites

9. **PWA polish** ‚Äî icons, manifest, offline caching
